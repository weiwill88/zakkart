# 工程日志：客户反馈修复与测试补齐

> 反馈来源：`erp系统PC&App端反馈20260422.docx`
> 执行日期：2026-04-30

## 11:00 左右｜反馈拆解与影响范围确认

- 将反馈归为四类：配件管理、合同生成/上传、司机 H5、小程序供应商/发货/库存。
- 已确认本轮不新增数据库集合，继续复用 `organizations / contracts / part_types / shipment_orders / users / inventory` 等现有集合。
- 当前工作区原本已有多处未提交改动，本轮按增量修改处理，没有回滚既有文件。

## 11:05 左右｜后端修复

- 修复配件管理批量编辑的唯一性口径：唯一性指纹从“类别 + 名称 + 供应商”扩展到“类别 + 名称 + 供应商 + 材质 + 颜色 + 尺寸”，避免同名但规格不同的配件被误判重复。
- 合同付款字段调整为：`supplier_bank_account_name` 表示户名，默认供应商公司名称；`supplier_bank_name` 表示开户银行；兼容旧合同字段。
- 合同交付计划增加强校验：每个采购货物的交付计划合计必须等于采购数量；一次性费用行不参与交付计划。
- 司机 H5 `ensureDriverObject` 改为只做内存归一化，避免对云数据库写入空对象导致 `$set is empty`。
- `order.generate` 生成合同时读取供应商 `contract_template.quality_clause`，支持按公司定制“成品货物要求”。

## 11:09 左右｜PC 端修复

- 合同正文中“采购数量”改为只录合计数量，不再展示 SKU 明细文本。
- 合同采购结算主表增加“一次性费用”行，费用计入总金额但不进入交付计划。
- 合同编辑、保存、推送确认、导出 Word、上传已签 PDF 前都接入交付数量一致性校验。
- 合同支持上传附录图纸；在线合同可预览，Word 导出会尽量嵌入可访问图片，无法嵌入时保留文件标识。
- 供应商银行信息主流程移除“开户行全称”输入；新增供应商合同模板字段，当前先支持“成品货物要求”。
- PC 云存储上传前强制确认匿名登录，并将 CloudBase storage 失败提示改得更明确。
- 库存新增表单默认进入“成品”库存，新增时不再展示在制品/半成品输入。

## 11:13 左右｜小程序与 H5 修复

- 供应商管理列表改为展示真实 `cooperation_status`，已停止合作的工厂不再显示“合作中”。
- 供应商成员无合同权限时，首页不再主动请求合同列表；后端也会阻断无合同权限的合同列表/详情/PDF 访问。
- 发货创建页对地址数据做 `id` 归一化，避免模板层直接依赖 `_id` 并减少 `document.get` 类报错扩散。
- 包装厂出库页补齐 `bindinput` 与表单校验，修复出库数量、目的地无法录入的问题；本轮不扩展完整阶段 5 出库后端闭环。
